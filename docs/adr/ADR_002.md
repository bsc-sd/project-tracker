
# ADR-002: Use React with TanStack Query for Frontend

## Status
Accepted

## Date
2024-07-01

## Context
We need a frontend framework that supports:
- Complex forms and data management
- Real-time data synchronization
- TypeScript for type safety
- Responsive design
- Fast development velocity

## Decision
Use **React 18** with **TypeScript**, **TanStack Query** for server state, and **Tailwind CSS** for styling.

## Rationale
- React has the largest ecosystem and community
- TanStack Query eliminates manual state management for server data
- TypeScript catches bugs at compile time
- Tailwind provides utility-first CSS with zero custom CSS files
- Vite offers instant HMR for fast development

## Alternatives Considered
| Technology | Reason Rejected |
|-----------|-----------------|
| Next.js | Over-engineered for SPA; SSR not needed |
| Vue.js | Smaller ecosystem for enterprise |
| Angular | Steeper learning curve, heavier bundle |
| Redux | TanStack Query handles server state better |

## Consequences
- Must manage client-side routing (React Router)
- Bundle size requires attention (code splitting)
- Team must understand React hooks patterns
