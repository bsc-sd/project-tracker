
# ADR-001: Use FastAPI with Async SQLAlchemy

## Status
Accepted

## Date
2024-07-01

## Context
We need a Python backend framework that supports:
- High performance for concurrent requests
- Automatic OpenAPI documentation
- Type safety with Pydantic
- Async database operations
- Production-ready with minimal boilerplate

## Decision
Use **FastAPI** with **SQLAlchemy 2.x (async mode)** and **asyncpg** driver.

## Rationale
- FastAPI is the fastest Python framework (comparable to Node.js/Go)
- Native async/await support for non-blocking I/O
- Automatic Swagger/ReDoc documentation from type hints
- Pydantic integration for request/response validation
- SQLAlchemy 2.x has first-class async support
- asyncpg is the fastest PostgreSQL driver for Python

## Alternatives Considered
| Framework | Reason Rejected |
|-----------|-----------------|
| Django REST Framework | Synchronous, heavier, slower |
| Flask | No native async, no auto-docs |
| Express.js (Node) | Client preferred Python ecosystem |

## Consequences
- Team must be comfortable with async/await patterns
- Some ORMs/libraries may not support async
- Slightly more complex testing setup (pytest-asyncio)
